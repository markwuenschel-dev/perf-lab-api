# Project Goal

## Project
**perf-lab-api** — A performance-science API that prescribes adaptive training loads using a multi-engine architecture (dose engine, state dynamics, constraint engine, prescriber).

## Current Phase
Phase 2 close-out: route tests, auth hardening, benchmark weak-point feedback.

## North Star
Ship a reliable, well-tested API that coaches can integrate with. Each agent's work should move the project closer to a production-ready v1.

## Success Criteria
- All routes have integration tests that pass against a real database.
- Auth is correct and auditable.
- Prescription logic is explainable and testable in isolation.
- Simulation scenarios exercise the full stack end-to-end.
- No agent loops forever; every turn produces a concrete artifact or a terminal decision.

## What This File Is
A stable anchor. Every agent reads this at session start and uses it to evaluate whether their output actually serves the goal. Do not edit this file unless the project's north star genuinely changes — in that case the Orchestrator updates it and logs the change in `/agent_collab/logs/decisions.md`.
