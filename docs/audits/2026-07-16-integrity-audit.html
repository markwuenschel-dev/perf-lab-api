Executive summary

The static integrity gates are green — ruff clean, pyright strict at 0 errors, 1,141 tests passing — and the repo has already absorbed two audit rounds. So the remaining findings are second-order: places where a committed write can still return a 500, where a hand-maintained list has no guard tying it to its filesystem, and where the model↔migration gate deliberately does not look. None of these are caught by the current CI.

Headline risk — C1
Wellness ingest can 500 on an already-committed write. After the wellness sample commits, three best-effort shadow writers run; any one failing calls db.rollback(), which expires the committed ORM object, and the endpoint's final model_validate(sample) then triggers a lazy load → MissingGreenlet → 500. The invariant "shadows never affect the response" is violated. The codebase already knows this hazard: state_service.py:1056-1058 guards the twin workout path with an explanatory comment; wellness.py:44 is missing that guard.

Six candidates are safe to auto-execute (contained, testable, no contract/migration/policy judgment). Seven require a human decision — chiefly a schema-drift cluster that needs migrations and backfills, plus three "wire-it or delete-it" product calls on flags and orphan modules. Auto mode executes the first group and parks the second.
