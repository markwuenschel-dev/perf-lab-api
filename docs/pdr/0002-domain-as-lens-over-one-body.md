# Domain is a lens over one body, not a mode switch

A "domain" (running, strength, gymnastics, …) is a **lens over one unified body
state**, never a separate mode the user toggles into. Consequences: the Twin shows all
axes always; readiness is whole-body and cross-talk-aware; blocks are hybrid by
default. Concurrent training is the first-class case, not an edge case.

We rejected per-domain isolated state / "modes" because it would fracture the single
`S(t)` the engine is built on and lose cross-talk.

**Guardrail:** do not introduce per-domain isolated state or a domain "mode switch" —
domain is a view, the body is singular.
